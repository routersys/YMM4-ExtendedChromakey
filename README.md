## 概要

本プラグインは YMM4 の「映像エフェクト」として動作する高精度クロマキーエフェクトです。
タイムライン上のアイテムに追加した「拡張クロマキー」ひとつで、緑・青・赤などの定型キーカラーだけでなく、任意のカスタムカラーをあらゆる色空間で指定してキーイングできます。
グラデーションによる範囲キーイング、スピル除去、エッジ品質調整、例外色による保護、残存色の色補正、前景の明るさ・コントラスト・彩度調整など、プロ品質の合成に必要な機能を網羅しています。
プロパティパネル最上部に常駐する検証パネルが、パラメータの矛盾や非効率な設定をリアルタイムで検出・通知するため、初心者から上級者まで安心して使用できます。

## インストール方法

1. [Releases](https://github.com/routersys/YMM4-ExtendedChromakey/releases/latest) ページから最新のプラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4 が起動していないことを確認し、ダウンロードしたファイルを実行（ダブルクリック）してインストールします。
3. YMM4 を起動し、タイムライン上のアイテムを選択します。
4. 映像エフェクトの追加から **「拡張クロマキー」** を選択してください。

## 主な機能

### 1. 多彩な色空間によるキーイング

「主要キーイング色」で **カスタム** を選択した場合、「色空間(カスタム用)」で色距離計算に使用する色空間を切り替えられます。用途に応じた最適な色空間を選ぶことで、難易度の高い素材にも対応できます。

| 色空間 | 特徴 |
|---|---|
| **RGB (標準)** | 標準的なRGB色距離。高速で汎用的 |
| **HSV (色相重視)** | 色相を軸に判定。色相範囲・彩度閾値を追加指定可能 |
| **CIE Lab (知覚的)** | 人の知覚に近い色距離。最もナチュラルな分離が可能 |
| **YUV (放送品質)** | 放送映像の標準色空間。明度範囲指定に対応 |
| **XYZ (CIE標準)** | CIE標準三刺激値による色距離計算 |
| **LCH (知覚円筒)** | Lab を円筒座標に変換。色相範囲指定に対応 |
| **CIEDE2000 (最高精度)** | 最高精度のΔE計算。最も緻密な色分離が可能 |

また、「主要キーイング色」で **緑 / 青 / 赤** を選択した場合は、プリセットベースの高速キーイングアルゴリズムが動作します。

### 2. グラデーションキーカラー

「カスタム」モードでは「基本色」と「終端色」の 2 色を指定し、「グラデーション強度」と「グラデーション角度」を組み合わせることで、画面上の位置に応じてキーカラーを変化させるグラデーションキーイングが可能です。照明ムラのある背景の除去に有効です。

- **基本色**：透過の基準となる起点カラー
- **終端色**：グラデーションの終点カラー。「グラデーション強度」が 0% の場合は無効
- **グラデーション強度**：グラデーション効果の強さ（0% で基本色のみ）
- **グラデーション角度**：グラデーションの方向（度数）

### 3. 品質最適化処理

エッジの品質をきめ細かく制御できる複数のフィルター処理を搭載しています。

- **エッジの柔らかさ**：透過境界のフェード幅を調整
- **クリップ(黒) / クリップ(白)**：マスクの黒・白レベルを調整してノイズ除去や前景の締まりを改善
- **エッジぼかし**：マスク境界をぼかして粗いエッジを滑らかに
- **エッジバランス**：キーイング境界を内側・外側に微調整（プラスで内側、マイナスで外側）
- **デスポット**：マスク内の小さなノイズ（穴・ゴミ）を除去
- **エッジの浸食**：マスク境界を拡大・縮小してフリンジを除去
- **エッジの色のじみ除去**：輪郭部分の色かぶりを低減してより自然な合成を実現
- **キー・クリーンアップ**：キーイングしたマスク境界を微調整（マイナスで縮小、プラスで拡大）
- **エッジ検出**：輪郭のディテールを保持する強度
- **ノイズ除去**：マスクから細かいノイズを除去
- **フェザリング**：マスクのエッジをぼかして背景と自然になじませる
- **スピル除去**：被写体への背景色の映り込み（色かぶり）を除去
- **半透明デスピル**：ガラス・煙・モーションブラーなど半透明部分のスピルを除去
- **透明度品質**：煙などの高透明度部分の処理品質を向上
- **アルファブレンド調整**：前景と背景の混合処理を微調整
- **輝度ミックス**：輝度情報をキーイング判定にどれだけ使用するかを調整
- **品質プリセット**：高速 / バランス / 高品質の 3 段階で処理品質を選択

### 4. 色置換

キーイングで除去した領域を指定した色で置き換えることができます。

- **置換色**：除去した領域を置き換える色（透明を指定すると不透明度を下げる効果）
- **置換の強度**：置換色で塗りつぶす強度
- **輝度保持**：色置換時に元の輝度をどれだけ保持するか

### 5. 例外設定

「例外設定」を有効にすることで、キーイングから除外したい色の範囲を保護できます。被写体に緑のロゴや青い衣装がある場合などに有効です。

- **例外色1 (基本)**：除外する基本色（透明の場合は機能しない）
- **例外色2 (終端)**：グラデーションの終点色（例外色1 と同様のグラデーション指定が可能）
- **許容値**：例外色の許容範囲
- **グラデーション強度**：例外色のグラデーション効果の強さ
- **グラデーション角度**：例外色のグラデーション方向

### 6. 色補正

「色補正」を有効にすることで、キーイング後の映像に対して色補正を適用できます。

- **補正対象色**：補正したい残存背景色
- **補正後の色**：残存色を変換する目標色
- **残存色補正**：クロマキーで残った背景色を別の色に変換する強度
- **補正許容値**：色補正の許容範囲（大きいほど広い範囲が補正される）
- **前景の明るさ**：キーイング後の前景の明るさ調整（-100% ～ +100%）
- **前景のコントラスト**：キーイング後の前景のコントラスト調整
- **前景の彩度**：キーイング後の前景の彩度調整

### 7. 完全クロマキーモード

「完全クロマキー」をオンにすると、輝度を保持せず指定色範囲を完全に透過させます。スピル除去・エッジ彩度除去・輝度保持なども自動的に無効化され、シンプルかつ確実な透過が得られます。

### 8. リアルタイムパラメータ検証

プロパティパネル最上部に常駐する検証パネル（`ParameterValidationPanel`）が、パラメータの組み合わせをリアルタイムで分析し、問題や注意点を自動的に通知します。メッセージが複数ある場合は、優先度の高いものから表示され、「+N」バッジで追加件数を示します。パネルにマウスを合わせると全メッセージのツールチップが表示されます。

| レベル | 色 | 内容 |
|---|---|---|
| **エラー** | 赤 | キーイングが正常に動作しない設定を検出 |
| **警告** | 黄 | 意図しない結果になる可能性がある設定 |
| **パフォーマンス** | 黄 | 処理負荷が高くなる組み合わせを検出 |
| **情報** | 青 | 設定の効果が無効になっているケースの通知 |
| **アップデート** | 緑 | 新しいバージョンが利用可能な場合に通知 |

### 9. 自動アップデート通知

プラグイン読み込み時に GitHub の最新リリースを確認し、新しいバージョンが公開されている場合は検証パネルに通知を表示します。通知はアップデート優先で表示され、他のメッセージより前面に出ます。

### 10. 多言語対応

YMM4 の言語設定に合わせて UI テキスト、グループ名、パラメータ名、バリデーションメッセージがすべて自動で切り替わります。

対応言語：日本語 (`ja-jp`)、英語 (`en-us`)、中国語簡体字 (`zh-cn`)、中国語繁体字 (`zh-tw`)、韓国語 (`ko-kr`)、スペイン語 (`es-es`)、アラビア語 (`ar-sa`)、インドネシア語 (`id-id`)

## バリデーションメッセージ一覧

プロパティパネル上部の検証パネルに表示されるメッセージの一覧です。

### エラー

| 条件 | メッセージ |
|---|---|
| カスタムモードで基本色が透明 | カスタムモードでは、基本色を不透明に設定してください。完全に透明な色はキーイングできません。 |
| クリップ(黒)の値がクリップ(白)以上 | クリップ(黒)の値がクリップ(白)以上になっています。マスクが正常に生成されません。 |

### 警告

| 条件 | メッセージ |
|---|---|
| 許容値が極端に低い（1%未満） | 許容値が極端に低いため、意図した範囲が透過されない可能性があります。 |
| 許容値が極端に高い（85%超） | 許容値が極端に高いため、前景の意図しない部分まで透過される可能性があります。 |
| エッジの柔らかさ > 0 かつ 許容値 < 1% | エッジの柔らかさを機能させるには、許容値を少し上げる必要があります。 |
| 輝度ミックスが高い（90%超） | 輝度ミックスが高すぎると、色の情報が無視され、明るさだけでキーイングされます。 |
| クリップとエッジの柔らかさが同時に高い | クリップ設定とエッジの柔らかさの両方が高く設定されています。意図しない硬い輪郭になる可能性があります。 |
| グラデーション強度が高い（90%超） | グラデーション強度が高すぎると、意図しない範囲まで透過される可能性があります。 |
| 例外色が透明かつ例外許容値 > 0 | 例外色が透明に設定されているため、「例外設定」は機能しません。 |

## 使い方ガイド

### 拡張クロマキーを追加する

1. タイムラインで映像アイテム（動画・画像など）を選択します。
2. 映像エフェクトの追加から「拡張クロマキー」を選択します。
3. プロパティパネルに検証パネルとパラメータ一覧が表示されます。

### グリーンバック（緑背景）をキーイングする

1. 「主要キーイング色」で **緑** を選択します。
2. 「許容値」を調整して透過範囲を広げます（目安: 20～40%）。
3. 「エッジの柔らかさ」を 5～15% 程度に設定してエッジを滑らかにします。
4. 「スピル除去」を 10～30% 程度に設定して被写体への緑の映り込みを除去します。
5. 必要に応じて「クリップ(黒)」「クリップ(白)」でマスクの黒白レベルを調整します。

### カスタムカラーをキーイングする

1. 「主要キーイング色」で **カスタム** を選択します。
2. 「基本色」で背景の代表色をスポイトで取得します（必ず不透明に設定）。
3. 「色空間(カスタム用)」を選択します（最初は **CIE Lab (知覚的)** を推奨）。
4. 「許容値」で透過範囲を調整します。
5. 照明ムラがある場合は「終端色」と「グラデーション強度 / 角度」を設定してグラデーションキーイングを有効にします。

### 被写体の特定の色を保護する（例外設定）

1. 「例外設定」の「有効」をオンにします。
2. 「例外色1 (基本)」で保護したい色を指定します（不透明に設定）。
3. 「許容値」で保護範囲を調整します。
4. 被写体に色のグラデーションがある場合は「例外色2 (終端)」と「グラデーション強度 / 角度」を活用します。

### キーイング後の色補正をする

1. 「色補正」の「有効」をオンにします。
2. 「前景の明るさ」「前景のコントラスト」「前景の彩度」で前景の色調を調整します。
3. クロマキー後に残った背景色がある場合は「補正対象色」にその色を指定し、「補正後の色」と「残存色補正」で修正します。

### デバッグモードを活用する

1. 「表示モード」を「マスク表示」に切り替えると、マスク（白: 前景 / 黒: 背景）をグレースケールで確認できます。
2. 「色距離」モードでは、キーカラーとの色距離をビジュアルで確認してパラメータ調整の参考にできます。
3. 「スピル除去後」「色補正後」モードで各処理段階の中間結果を確認できます。
4. パラメータ調整が完了したら「最終結果」に戻してください。

### 検証パネルのメッセージを解消する

1. プロパティパネル上部に表示されているメッセージの内容を確認します。
2. 赤（エラー）のメッセージは必ず解消してください。正常にキーイングできません。
3. 黄（警告）のメッセージはケースによっては意図通りの設定の場合もありますが、内容を確認してください。
4. 青（情報）のメッセージは設定の効果が無効になっているケースの通知です。不要なパラメータを初期値に戻すと解消されます。
5. メッセージにマウスを合わせると全メッセージのツールチップが表示されます（「+N」バッジがある場合）。

## 注意事項

- AviUtl 形式（`.exo`）への出力には対応していません。
- 「完全クロマキー」をオンにすると、スピル除去・エッジぼかし・エッジバランス・デスポット・エッジの浸食・エッジの色のじみ除去・キー・クリーンアップ・エッジ検出・置換色関連・半透明デスピルが無効になります。
- カスタムモードで「基本色」を透明（アルファ 0）に設定した場合、キーイングは実行されません。検証パネルにエラーが表示されます。
- 「クリップ(黒)」の値が「クリップ(白)」以上になるとマスクが正常に生成されません。検証パネルにエラーが表示されます。
- 「品質プリセット」を「高品質」に設定した状態で「エッジぼかし」「デスポット」など重い処理を組み合わせると、処理負荷が大きくなります。プレビュー中のパフォーマンスに影響する場合は「高速」または「バランス」をお試しください。
- 「CIEDE2000 (最高精度)」は最も精度が高い反面、計算コストも最大です。
- 「色相範囲」は色空間が「HSV」または「LCH」の場合のみ有効です。「彩度閾値」は「HSV」のみ有効です。「明度範囲」は「RGB」「YUV」のみ有効です。
- 「スピル除去」は「主要キーイング色」が「緑」「青」「赤」のときに最も効果的です。カスタムモードでも動作しますが、効果が限定的な場合があります。
